4. Call Routing & Inbound Architecture

VSP-VOIP uses two parallel inbound paths on Telnyx. Configuration is on the Call routing page (/greeting).

PSTN Caller │ ▼ Telnyx Phone Number (assigned to tenant) │ ├── Call Control App ──► POST /webhook/call-control │ └── lib/inboundCallControl.js │ • Business hours check │ • IVR (gather_using_speak) │ • Ring group → PSTN + WebRTC (dial to sip:user@sip.telnyx.com) │ • Voicemail (record) │ • Call recording │ └── TeXML Connection ──► POST /webhook └── lib/callRouting.js + lib/texml.js • Greeting (Say/Play) • IVR (Gather) • Forward/Dial • Voicemail • Recording notice

Routing configuration (Greeting model)

SettingDescription
Greeting message / audioPlayed on answer (TTS or uploaded MP3/WAV)
Business hoursTimezone + weekly schedule; after-hours message
IVR menuDTMF options → forward, voicemail, ring group, etc.
Ring groupMembers: PSTN numbers or app users (WebRTC SIP)
Ring strategySimultaneous or sequential; timeout seconds
VoicemailEnable, prompt, max length, email notification
Call recordingInbound recording notice + storage
Per-number routingOverride on /my-numbers (forward, IVR, etc.)

Important: Inbound WebRTC ring groups require the Call Control path. TeXML alone cannot dial WebRTC clients.

5. WebRTC Softphone (Web)

Page: /softphone · SDK: @telnyx/webrtc

  1. User opens softphone → frontend calls GET /api/softphone/config.
  2. Backend creates/retrieves per-user Telnyx telephony credential → POST /api/softphone/token.
  3. Browser SDK connects with login token (24h expiry).
  4. Presence heartbeat every ~60s keeps user "online" for ring groups.
  5. Outbound: dial E.164, optional recording via POST /api/softphone/record-start.
  6. Hangup logs call via POST /api/softphone/call-log.

Credentials stored on User: telnyxCredentialId, telnyxSipUsername, softphoneOnlineAt.

6. SMS

Two-way SMS via Telnyx Messaging API. Requires messaging profile assigned to tenant numbers.

  • Outbound: POST /api/sms/send → Telnyx POST /v2/messages
  • Inbound: Telnyx webhook POST /webhook/sms → saved to SmsMessage
  • Delivery status: Updated via same webhook (message.finalized events)
  • Webhook requirement: API_PUBLIC_URL must be a public HTTPS URL (not localhost)

7. Numbers & Billing

Number purchase flow

  1. Search numbers at /numbers (Telnyx GET /available_phone_numbers).
  2. Add to cart → checkout with Stripe or manual bank transfer.
  3. Stripe: webhook completes order → auto-fulfill via Telnyx POST /number_orders.
  4. Manual: super admin fulfills from admin orders page.
  5. Purchased numbers assigned TeXML connection + messaging profile automatically.

Fee structure

Per-tenant fees from Tenant model: platformFeeSetup, platformFeeMonthly, platformFeeFirstMonth. Telnyx carrier costs are separate.

See also: Number Purchase & Billing Documentation

8. Common User Workflows

8.1 Tenant admin: set up call routing

  1. Go to Call routing → configure greeting and business hours.
  2. Add ring group members (team users with softphone + optional PSTN numbers).
  3. Enable voicemail and recording as needed.
  4. Assign numbers to users on My Numbers.
  5. Test inbound call to verify ring group reaches online softphone users.

8.2 Tenant user: make an outbound call

  1. Open Softphone → wait for "Connected" status.
  2. Select caller ID from tenant numbers.
  3. Dial destination → talk → hang up.
  4. View call in Call History.

8.3 Tenant admin: send SMS

  1. Open SMS → select or start conversation.
  2. Choose sending line (tenant DID) → type message → send.
  3. Inbound replies appear via Telnyx webhook (requires public API URL).
