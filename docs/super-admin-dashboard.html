1. Executive summary — Is everything dynamic?

Short answer: No — not everything is fully dynamic. Most implemented Super Admin screens load live data from PostgreSQL via the Express API. Several telecom KPIs use estimates or heuristics, some nav items are marked Soon, two pages are pure placeholders, and two nav links return 404.

3. Authentication & role gating

LayerBehavior
Backend (authoritative) Every /api/admin/* route uses authMiddleware + requireRole('SUPER_ADMIN'). Returns 403 if role does not match.
Login redirect SUPER_ADMIN → /admin. Other roles → /dashboard.
Per-page frontend Functional pages call getMe() and redirect non–super-admins to /dashboard; 401 → /login.
App layout Requires authentication only — no role check at layout level.
Placeholder pages /admin/voice-quality and /admin/security do not enforce SUPER_ADMIN on the frontend (static content only).
Super admin with tenant If user has tenantName, sidebar shows compact admin links instead of full AdminNav.

7. All admin pages — detailed breakdown

/admin/tenants — Company directory

web/src/app/(app)/admin/tenants/page.tsx · Fully dynamic

APIs: GET /tenants, POST /tenants, POST /tenants/cleanup-empty, PUT /tenants/:id/status

DataTable columns: Tenant name, Status, Setup/number, First month, Recurring/mo, Numbers, Users, Actions (Suspend, Manage).

Create tenant: Slide-over panel → POST with name and fee fields. Form defaults (e.g. monthly $8) are UI defaults only.

/admin/tenants/[id] — Tenant detail

web/src/app/(app)/admin/tenants/[id]/page.tsx · Fully dynamic

Sections: Organization name (editable), Platform fees (setup/first/recurring), Suspend/activate, Users DataTable, Phone numbers DataTable, Add user form.

API data not shown in UI: _count.callLogs, Stripe subscription IDs (returned by API but not displayed).

/admin/orders — Orders & invoicing

web/src/app/(app)/admin/orders/page.tsx · Fully dynamic

API: GET /orders?status=&paymentMethod= (max 100 rows). Default filter: PENDING.

DataTable: Date, Tenant, Payment method, Invoice #, Total, Status, Reference.

/admin/orders/[id] — Order detail

web/src/app/(app)/admin/orders/[id]/page.tsx · Fully dynamic

Actions: Edit payment reference, Send invoice (bank), Fulfill (Telnyx purchase), Cancel. Stripe orders show explanatory static copy.

/admin/revenue — Revenue & tariffs

web/src/app/(app)/admin/revenue/page.tsx · Fully dynamic

API: GET /revenue — sums totalCharged on FULFILLED/PAID/PARTIAL orders.

Widgets: Total revenue + order count cards; DataTables by tenant and by month. No tariff editor on this page (tariffs in settings + per-tenant billing).

/admin/audit — Forensic audit trail

web/src/app/(app)/admin/audit/page.tsx · Fully dynamic

API: GET /audit-log?limit=100

DataTable: When, Admin email, Action, Entity type/id, Details (JSON).

/admin/inventory — DID number pool

web/src/app/(app)/admin/inventory/page.tsx · Partially dynamic

API: Reuses GET /tenants — no dedicated inventory endpoint.

  • Dynamic: Assigned DIDs (sum of tenant numberCount), active tenants with numbers, tenant allocation table.
  • Static: "Wholesale pool" KPI shows hardcoded label "Telnyx" with "External" badge.
  • Missing: Unassigned DID pool count (availableDidPool is null in dashboard API and unused).

/admin/settings — Platform settings

web/src/app/(app)/admin/settings/page.tsx · Fully dynamic

APIs: GET/PUT /platform-settings, GET /telnyx/status

Sections: Stripe keys (masked on load), Webhook URL (server-computed), Bank transfer fields, Telnyx connection IDs, Telnyx live status panel, Default VSP markup fees.

Static UI only: Example pricing math uses hardcoded carrier $1.25 in help text; Stripe dashboard external link.

/admin/voice-quality — Voice quality & heatmaps

web/src/app/(app)/admin/voice-quality/page.tsx · Not dynamic

Uses AdminPlaceholderPage — static "Coming soon" roadmap. No API calls. Nav badge: Soon.

/admin/security — RBAC & compliance

web/src/app/(app)/admin/security/page.tsx · Not dynamic

Uses AdminPlaceholderPage — static "Coming soon" roadmap. No API calls. No role management UI exists yet.

11. Known gaps & recommendations

  • Replace estimated KPIs — Wire MOS and SIP registration to Telnyx/webhook telemetry or a monitoring service.
  • LNP workflow — Build porting page + API; replace hardcoded pendingLnpRequests: 0.
  • Inventory API — Add wholesale pool count from Telnyx number search or inventory table.
  • Carrier balance — Integrate Telnyx billing/balance API for live spend vs balance KPI.
  • Fix dead nav links — Create pages for /admin/inventory/porting and /admin/trunking/lcr, or remove links until ready.
  • Resource quotas — Separate page + API for per-tenant limits (users, DIDs, concurrent calls).
  • RBAC UI — Role management beyond binary SUPER_ADMIN / TENANT_ADMIN.
  • Placeholder page auth — Add SUPER_ADMIN gate to voice-quality and security pages for consistency.
