Platform Overview

VSP Phone is a multi-tenant cloud VoIP platform that delivers business voice, SMS, voicemail, call recording, and mobile softphone capabilities powered by Telnyx carrier infrastructure.

What is VSP Phone?

VSP Phone is a RingCentral-style UCaaS platform. A platform operator (Super Admin) provisions tenant organizations. Each tenant receives phone numbers, extensions, users, call routing, billing, and mobile access through a unified web and mobile experience.

Production URLs: https://vspphone.com (landing) · https://app.vspphone.com (tenants) · https://admin.vspphone.com (super admin) · https://api.vspphone.com (API)

Telnyx Integration

All voice and SMS traverse Telnyx CPaaS: number search/purchase, Call Control webhooks, WebRTC credentials, SMS messaging profiles, and call recordings sync from Telnyx storage.

Number Management

Numbers are searched via Telnyx inventory, purchased through the tenant cart or super-admin purchase flow, assigned to tenants/extensions, and routed via IVR, ring groups, or direct extension targets.

Billing System

Dual billing: Stripe card payments and manual bank transfer with admin approval. Per-number monthly fees, platform fees, and order invoices are tracked in PostgreSQL.

Number Management (Super Admin)

Complete Workflow

  1. Search Numbers — Query Telnyx inventory by area code, city, or pattern at /admin/numbers/purchase.
  2. Purchase Numbers — Add to cart; checkout via Stripe or platform-paid order; assign to target tenant.
  3. Assign Numbers — Link DID to tenant, extension, ring group, or routing rule.
  4. Release Numbers — Return number to carrier inventory; update tenant billing.
  5. Port Existing Numbers — Submit LOA and port request (see Porting section).

Number Porting (LNP)

What is Number Porting?

Number porting (Local Number Portability) transfers an existing phone number from a customer's current carrier to VSP Phone / Telnyx without changing the published number.

Porting Process

  1. Customer submits LOA — Letter of Authorization with account number and numbers to port.
  2. Billing verification — Confirm tenant account is in good standing.
  3. Carrier validation — Telnyx validates number eligibility and losing carrier.
  4. FOC date received — Firm Order Commitment date scheduled.
  5. Number migration — Cutover window; routing updated on FOC.
  6. Testing — Inbound/outbound test calls on ported DID.
  7. Completion — Status set to Completed; number active in tenant inventory.

Status Values

StatusMeaning
SubmittedRequest received, awaiting review
Pending ReviewAdmin or carrier reviewing LOA documents
RejectedInvalid LOA, wrong account info, or carrier denial
ApprovedAccepted by carrier; awaiting FOC
In ProgressPort order active with losing carrier
FOC ReceivedFirm Order Commitment date confirmed
CompletedNumber live on VSP Phone platform

Risks & Validation Checks

  • LOA signature and authorized contact must match carrier records
  • Account PIN / BTN must be correct or port will reject
  • Numbers must not have pending port with another carrier
  • Toll-free and wireless numbers have additional requirements
  • Partial port failures require per-number status tracking

Mobile Application

The VSP Phone Android app provides WebRTC softphone, SMS, voicemail, and call history for mobile workers.

Login & Provisioning Process

  1. Tenant admin creates user — Add user at Settings → Team
  2. Extension assigned — Link user to extension with SIP/WebRTC credentials
  3. User receives credentials — Email/password or QR provisioning token
  4. Install VSP Phone app — Download APK from landing page or Play Store (when published)
  5. Login — Email + password against https://api.vspphone.com
  6. Extension config download — App fetches softphone config and Telnyx token
  7. WebRTC registration — SIP registration with Telnyx completed
  8. Ready — User can make and receive calls with push notifications
QR provisioning: Admin generates token at extension → user scans in app for zero-touch setup.
