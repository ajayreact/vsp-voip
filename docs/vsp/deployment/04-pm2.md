# PM2 — Next.js Web Portal

The tenant and super-admin portals are a single Next.js app managed by **PM2** on EC2. Nginx proxies `app.vspphone.com` and `admin.vspphone.com` to port **3001**.

Config: [deploy/pm2.ecosystem.config.js](../../../deploy/pm2.ecosystem.config.js)  
Deploy script: [deploy/deploy-web.sh](../../../deploy/deploy-web.sh)

---

## Process details

| Setting | Value |
|---------|-------|
| App name | `vsp-web` |
| CWD | `./web` |
| Command | `npm start` (Next.js production server) |
| Port | 3001 |
| `NODE_ENV` | `production` |
| `NEXT_PUBLIC_API_URL` | `https://api.vspphone.com` |

---

## First-time setup

```bash
cd /opt/vsp-voip
bash deploy/deploy-web.sh
# Or manually:
pm2 start deploy/pm2.ecosystem.config.js
pm2 save
pm2 startup   # follow printed instructions for boot persistence
```

---

## Daily operations

```bash
pm2 status vsp-web
pm2 logs vsp-web
pm2 logs vsp-web --lines 100
pm2 restart vsp-web
pm2 stop vsp-web
pm2 save
```

After every production web deploy:

```bash
bash deploy/deploy-web.sh   # includes restart + save
```

---

## Deploy without the script

If you must deploy manually, **always build before restart**:

```bash
cd /opt/vsp-voip/web
rm -rf node_modules .next
npm ci
NEXT_PUBLIC_API_URL=https://api.vspphone.com npm run build
cd ..
pm2 restart vsp-web
pm2 save
```

**Never** `pm2 restart` without a fresh `npm run build` when source changed — users will run old `.next` output or crash if `node_modules` is inconsistent.

## Verify PM2 health

```bash
pm2 describe vsp-web
curl -sI http://127.0.0.1:3001/ | head -5
curl -sI https://app.vspphone.com/ | head -5
```

Check for crash loops:

```bash
pm2 logs vsp-web --err --lines 50
```

Common errors:

- `next: command not found` — run `npm ci` in `web/`
- Port 3001 in use — `ss -tlnp | grep 3001`
- Build missing — no `.next/BUILD_ID` after deploy

---

## PM2 restart forgotten

Symptom: `git pull` and build succeeded but browser still shows old UI.

Fix:

```bash
pm2 restart vsp-web && pm2 save
```

Also hard-refresh browser (Ctrl+Shift+R) or test in incognito — see [11-known-issues.md](./11-known-issues.md).
