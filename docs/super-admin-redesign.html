VSP-VOIP Super Admin Portal — Product Audit & Redesign

Role: Senior SaaS Product Architect & UX Designer  |  Scope: Super Admin portal only  |  Goal: Enterprise-grade admin for 10,000+ tenants  |  Status: Recommendations — no code yet

Executive Summary

The current Super Admin portal is functionally rich but structurally fragmented. It has 16 routes across 7 sidebar sections, with overlapping KPIs, split settings forms, a misleading inventory page, and two different navigation patterns depending on whether the super admin has a tenant context.

Redesign outcome: Consolidate 16 pages → ~12 primary destinations organized by business function. Separate Executive Dashboard from Live Operations. Introduce net-new modules (Active Calls, Usage Monitoring, Support Tickets, Subscription Plans) requiring backend investment.

Current State

  • 7 sidebar sections, 14 nav items
  • Settings split across 4 pages
  • Engineering + executive KPIs mixed
  • No real-time active calls view
  • Fake "inventory" page

Target State

  • 7 top-level business areas
  • Unified Settings hub (tabbed)
  • Executive-first dashboard
  • Live Operations drill-down
  • Real telecom number inventory

Design references: RingCentral Admin Portal, Dialpad Office Admin, OpenPhone Workspace Admin, Aircall Dashboard, Zoom Phone Management, GoTo Admin Center.

19. Top 10 Priorities

  1. Unify navigation — one admin shell; remove sidebar admin shortcuts
  2. Split dashboard — Executive vs Live Operations (biggest UX win)
  3. Merge settings — 4 pages → 1 tabbed Settings hub
  4. Replace fake inventory — real number lifecycle (purchase/assign/release)
  5. Build Active Calls — Redis sessions + admin API
  6. Add global search — mandatory before 500+ tenants
  7. Usage Monitoring — per-tenant cost dashboard
  8. Introduce SubscriptionPlan — Starter/Growth/Business/Enterprise
  9. Pre-aggregate metrics — required for executive charts at scale
  10. Fix KPI link bugs — SIP card should not link to Stripe
