1. Overview & Access

The Super Admin portal is the platform operator control plane. It is separate from tenant-facing features and requires the SUPER_ADMIN role.

ItemDetail
URL prefix/admin (Next.js, port 3001)
Login redirectSUPER_ADMIN users land on /admin after login
API prefix/api/admin/* (Express, port 3000)
AuthJWT Bearer token + requireRole('SUPER_ADMIN')
Nav componentweb/src/components/admin-nav.tsx
Route moduleroutes/admin.js

Access modes

  • Super admin without tenant — sidebar shows only admin navigation (full operator view).
  • Super admin with tenant — tenant nav plus compact admin links at bottom of sidebar (can operate as tenant user too).

5. Telnyx Configuration (Super Admin)

Super admins configure Telnyx at Admin → Platform settings (/admin/settings). Values are stored in the PlatformSettings singleton and used across the platform.

SettingTelnyx resourcePurpose
telnyxConnectionIdTeXML Application / SIP ConnectionInbound PSTN via TeXML webhooks at /webhook
telnyxCredentialConnectionIdCredential ConnectionWebRTC softphone auth; webhook at /webhook/voice
telnyxMessagingProfileIdMessaging ProfileSMS send/receive; webhook at /webhook/sms
telnyxCallControlApplicationIdCall Control ApplicationInbound Call Control; webhook at /webhook/call-control

On API startup, the server auto-configures Telnyx webhooks when API_PUBLIC_URL is set (see lib/telnyxRecordingSetup.js, lib/telnyxMessagingSetup.js, lib/telnyxCallControlSetup.js).

6. Key Workflows

6.1 Onboard a new tenant

  1. Go to Company directory → Create tenant (name, contact email).
  2. Optionally create a TENANT_ADMIN user with email/password.
  3. Set per-tenant billing fees and quotas on tenant detail page.
  4. Tenant admin logs in at /login and purchases numbers or admin fulfills a manual order.

6.2 Fulfill a manual bank order

  1. Tenant places manual bank order from web portal.
  2. Super admin opens Orders → order detail.
  3. Mark payment received, then Fulfill → calls Telnyx POST /number_orders.
  4. Numbers assigned to tenant with TeXML connection + messaging profile.

6.3 Monitor platform health

  1. System vitals — check Telnyx connectivity badge.
  2. Voice quality — review MOS/jitter trends.
  3. Audit trail — review recent admin actions.
