VSP-VOIP Platform Audit & Architecture Report

Document Type: Software Architecture Assessment  |  Audit Date: June 21, 2026  |  Scope: Full codebase — backend, web, mobile, database, integrations

1. Executive Summary

VSP-VOIP is a functional UCaaS MVP built as a monolithic Node.js/Express backend with a Next.js tenant + super-admin portal, a Flutter mobile app, and Telnyx as the sole voice/SMS carrier. The platform delivers real voice calling (inbound/outbound WebRTC), SMS, voicemail, call recording, IVR/ring groups, number purchasing, and dual billing (Stripe + manual bank transfer).

It is not yet production-ready at scale — infrastructure, automated testing, security hardening, and enterprise UCaaS features are largely absent.

12. Scalability Assessment

Scale TargetFeasibilityBottlenecks
100 tenantsLikely OKSingle Express instance sufficient
1,000 tenantsRiskyIn-memory sessions, tenant cache
10,000 tenantsNot readyMonolith, no webhook queue, table growth
100,000 usersNot readyPer-user WebRTC credentials, SIP polling

Primary Bottlenecks

  1. Single-process Express with in-memory state
  2. Synchronous Telnyx webhook processing — no retry queue
  3. PostgreSQL — no read replicas or partitioning
  4. Voice telemetry polling hits Telnyx API on interval
  5. Local file storage for greetings — not shared across instances

19. CEO Summary

1. What has already been built?

VSP-VOIP is a working multi-tenant UCaaS MVP with a full web portal (tenant + super admin), Flutter mobile app with outbound/inbound WebRTC calling, Telnyx-powered voice (IVR, ring groups, voicemail, recording), SMS, phone number purchasing with Stripe/bank billing, and platform admin for tenants, orders, revenue, and voice quality. Equivalent to ~6–9 months of focused development.

2. What is production ready?

Functionally ready for controlled pilot: core voice, SMS, number provisioning, tenant onboarding by super admin, basic call routing and voicemail. Not operationally production ready without security fixes, deployment automation, and monitoring.

3. What is missing?

Enterprise telephony (transfer, conference, queues, extensions), self-service signup/trials/plan tiers, production infrastructure, security hardening, automated testing, E911, SSO, contact center, video, integrations, and mobile gaps (voicemail screen, SMS compose).

4. Can we onboard paying customers today?

Yes — for a manual pilot only. Super admin can create tenants, assign numbers, and collect payment. Suitable for 5–20 friendly customers with hands-on support. Not recommended for public self-service or enterprise without Phase 1 work.

5. Can this scale to 10,000 tenants?

Not in current architecture. With Phase 1–3 investments, 1,000 tenants is achievable; 10,000 requires Phase 3–4 architecture (horizontal scaling, webhook queues, DB optimization).

6. Estimated timelines

MilestoneTimeline
Pilot launch (5–20 customers)8–12 weeks
Commercial SMB launch (100+ tenants)5–7 months
Enterprise launch (SSO, E911, compliance)12–18 months

7. Bottom line

VSP-VOIP has a solid telephony foundation comparable to an early-stage RingCentral competitor for SMB voice + SMS (~30–35% feature parity). The gap to commercial and enterprise readiness is primarily in operations, security, scale architecture, and advanced UCaaS features — not in the core voice stack.
