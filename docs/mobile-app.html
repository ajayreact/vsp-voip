1. Overview & Development Phases

The mobile app is a tenant-only client. It consumes the same Express REST API as the Next.js web portal. Super admin features are not available on mobile.

5. WebRTC Softphone

Controller: mobile/lib/features/softphone/providers/softphone_controller.dart

Outbound call flow

  1. Open Phone tab → loads config + requests login token.
  2. TelnyxClient connects with token from POST /api/softphone/token.
  3. User selects caller ID (tenant DID), enters E.164 destination.
  4. Grant microphone permission → tap Call.
  5. Controls: mute, speaker, hang up.
  6. Optional: start recording via POST /api/softphone/record-start.
  7. On hangup: log call via POST /api/softphone/call-log.

Presence for ring groups

Heartbeat to POST /api/softphone/presence every ~60 seconds. Server marks user online if softphoneOnlineAt is within 5 minutes — required for inbound ring group delivery.

6. Inbound Calling & Push Notifications

Phase 3 enables receiving PSTN calls on the mobile softphone when the app is backgrounded or killed.

Components

ComponentLocationPurpose
Push coordinatorcore/push/push_call_coordinator.dartCoordinates push → SDK wake → call UI
Native incoming UIcore/push/native_incoming_call_ui.dartAnswer/Decline native UI
Firebase MessagingAndroidFCM push delivery
CallKitiOSflutter_callkit_incoming native call UI
ConnectionServiceAndroidSystem-level call integration
Push metadataSDKTelnyxClient.setPushMetaData()

Inbound call flow

  1. PSTN caller dials tenant number → Telnyx Call Control routes to ring group.
  2. Call Control dials WebRTC destination: sip:{sipUsername}@sip.telnyx.com.
  3. If app offline: Telnyx sends push via FCM (Android) or VoIP push (iOS).
  4. App wakes → registers push token (POST /api/softphone/push-token) → SDK receives invite.
  5. Native Answer/Decline UI → on answer, active call in softphone screen.

Setup requirements

  • Telnyx Credential Connection with push credentials configured in Telnyx Portal.
  • Firebase project for Android (FCM).
  • APNs VoIP certificate for iOS.
  • User must be ring group member in tenant call routing config.
  • See: mobile/docs/inbound-calling-setup.md

7. Setup & Run

Prerequisites

  • Flutter SDK 3.16+ (flutter doctor)
  • Android Studio and/or Xcode
  • API running: npm run dev:api from repo root
  • Seeded tenant user in PostgreSQL

Commands

cd mobile
flutter pub get

# Android emulator
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:3000

# Physical device on same Wi-Fi
flutter run --dart-define=API_BASE_URL=http://192.168.x.x:3000

# iOS simulator
flutter run --dart-define=API_BASE_URL=http://localhost:3000
