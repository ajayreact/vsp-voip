1. Overview — What You Are Building

A multi-tenant cloud phone platform (like RingCentral / Zoom Phone) where:

  • You (Platform Owner) operate VSP-VOIP and onboard business customers (tenants).
  • Each Tenant (e.g. Asuitech Solutions) gets phone numbers, users, greetings, call logs, and optional mobile apps.
  • Telnyx is the carrier layer (PSTN, SIP, TeXML / Voice API). Your software is the control plane.
  • Web app = admin + tenant portals. Flutter app = softphone / call history for end users (built by another developer using your API).

9. End-to-End Call Flow (Current + Future)

9A. Inbound Call Today (Working)

Caller dials +1 (309) 988-0196 │ ▼ Telnyx PSTN │ ▼ POST /webhook VSP-VOIP Server │ ├─ Lookup PhoneNumber → Tenant (Asuitech) ├─ Build TeXML: "Welcome to Asuitech Solutions..." └─ Log CallLog │ ▼ Caller hears greeting → line open 60 sec → hang up

9B. Inbound Call with Web Portal (Phase 3)

Caller dials tenant number │ ▼ Telnyx → POST /webhook │ ▼ Server reads Greeting + IvrMenu from DB (edited in web UI) │ ├─ After hours? → voicemail TeXML ├─ IVR Press 1 → Dial extension └─ Default → greeting + queue / forward

9C. Outbound from Flutter App (Phase 5)

User taps Call in Flutter app │ ▼ POST /api/calls/outbound │ ▼ Server → Telnyx Voice API / WebRTC token │ ▼ Flutter WebRTC SDK ↔ Telnyx ↔ PSTN

13H. Architecture Decision (Telnyx Email — Finalized)

Telnyx compared three UCaaS architectures. VSP-VOIP uses Option 3 (Pure Telnyx API). You are already on the correct path.

OptionWhat it isVSP-VOIP?
1. FusionPBX + FreeSWITCH Open-source PBX GUI + media server; Telnyx via SIP trunk No — you build the portal yourself
2. FreeSWITCH + OpenSIPS/Kamailio Custom SIP proxy + media; Telnyx via SIP trunk No — high complexity, SIP expertise
3. Pure Telnyx API + WebRTC TeXML / Call Control API + Numbers API; no servers to run YES — current stack

How Telnyx products map to Option 3 (your stack)

Telnyx productYour useDoc
Phone Numbers APISearch, buy, assign numbersNumbers
TeXML ApplicationInbound greeting, IVR (Phase 1–4)TeXML
Voice API + WebRTCFlutter softphone (Phase 6)Voice API
REST API v2All backend callsAPI Fundamentals
SIP TrunkingNot needed now — only if you add FreeSWITCH laterSIP Trunking

SIP Trunking vs TeXML (important):

  • SIP Trunking = connect a PBX (FreeSWITCH, Asterisk, FusionPBX) to Telnyx for PSTN. You manage SIP servers.
  • TeXML / Voice API = Telnyx runs call logic; your server returns XML or JSON commands. You chose this.
  • Both can buy numbers from Telnyx — you use the Numbers API, not SIP credentials.

When to reconsider Options 1 or 2

Only if you later need on-prem SIP, custom codec processing, or 100k+ concurrent calls on your own media servers. For a new 2026 cloud UCaaS with Flutter apps, Telnyx recommended Option 3 — matches your plan.
